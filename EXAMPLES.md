# 🏷️ 标签书签示例

本文档展示如何为书签添加标签，让你更好地组织和管理浏览器书签。

## 📌 标签格式规则

在书签的**名称**后面添加 `#标签名`，多个标签用空格分隔：

```
书签名称 #标签1 #标签2 #标签3
```

## 💡 标签分类建议

### 按功能分类
```
#开发 - 开发工具和资源
#设计 - 设计工具和资源
#工作 - 工作相关
#学习 - 学习资源
#参考 - 参考文档
```

### 按技术栈分类

## ✏️ 修改书签标签

### 方法：Chrome 书签管理器
1. 按 `Ctrl+Shift+B`（或 `Cmd+Shift+B` 在Mac上）打开书签栏
2. 右键点击书签 → 编辑
3. 在名称字段修改标签部分
4. 点击保存

### 修改示例

**修改前:**
```
Google #搜索
```

**修改后：**
```
Google #搜索 #工具 #常用
```

## 📊 标签使用统计建议

推荐常用标签数量：**5-15个**

- **太少** (1-3个): 分类不足，难以快速筛选
- **适中** (5-15个): 方便记忆和筛选 ✅
- **太多** (20+个): 影响性能和可维护性

## 🔍 搜索技巧

在插件顶部搜索框输入内容，可以：

1. **搜索标签** - 输入 `#开发` 或 `开发`
2. **搜索书签** - 输入 `GitHub` 会匹配 `GitHub #开发`
3. **搜索URL** - 输入 `github.com` 会找到所有GitHub书签
4. **组合搜索** - 搜索适用于标题、标签和URL

## ✨ 最佳实践总结

| 实践 | 示例 | 优势 |
|------|------|------|
| ✅ 保持一致 | 所有开发工具都用 `#开发` | 易于维护和搜索 |
| ✅ 多标签组合 | `#开发 #前端 #JavaScript` | 提供多维度分类 |
| ✅ 简短标签名 | `#开发` 而不是 `#web开发工具` | 易于输入和记忆 |
| ✅ 定期审查 | 每月检查一次标签使用情况 | 保持标签系统整洁 |
| ❌ 避免重复 | 不要用 `#dev` 和 `#开发` | 易引起混淆 |
| ❌ 避免冗长 | 不要用超长的标签名 | 影响用户体验 |
